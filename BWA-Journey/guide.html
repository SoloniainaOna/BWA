Internal reference
Managing
the journey
This guide covers everything needed to run the BWA Journey experience — from setup before visitors arrive to resetting between sessions. Keep this page open on a staff tablet throughout the event.
1
Experience Overview

Visitors follow a four-step guided journey, all managed from a single tablet. The flow is linear — each step must be completed before advancing to the next. No staff intervention is required during a normal visit.

# Step Duration Location
1 Explore the Space Walk the room, discover works, experiment with interactive elements. 15 – 20 min Entire room
2 Digital Experiences Three interactive works on the tablet: ripple, filters, puzzle. 20 – 30 min On tablet
3 Virtual Reality VR headset experience. Staff-assisted if needed. 10 – 15 min VR station
4 Companion App Open the third-party app to complete the journey. 5 – 10 min On tablet
2
Before Visitors Arrive

Run through this checklist before each session opens.

  • All visitor tablets are charged to at least 80% and open on index.html — the Welcome screen.
  • VR headsets are charged, sanitised, and placed at the VR station.
  • The BWA Experiences app (tablet experiments) is accessible — test the link in Step 2.
  • The companion app URL (Step 4) is correctly configured in index.html.
  • Room lighting and all interactive physical installations are active and functional.
  • Wi-Fi is stable. Test loading the experiences app and companion app from a visitor tablet.
  • All tablets are set to auto-brightness off, screen timeout to Never.
3
Step-by-Step Reference
Step 01 — Explore
Explore the Space
Visitors are free to walk the room. No action required from staff unless a visitor has a question. The tablet shows them a list of what to try. Ensure all interactive physical elements are working before they enter. The visitor advances by tapping Finished on the tablet.
Step 02 — Digital
Digital Experiences
The tablet links directly to the BWA Experiences app. There are three works: Ripple & Trail, Filters, and Puzzle. Visitors can try them in any order and return at any time. If the link does not open, check that the experiences server is running or that the relative path is correctly set (see Configuration below). When done, visitor taps Finished to proceed.
Step 03 — VR
Virtual Reality
The visitor leaves their tablet at a table and picks up a VR headset. Staff should be available near the VR station to help with headset fitting if needed. Remind visitors to leave the tablet somewhere safe before putting on the headset. After the VR session, they return to the tablet and tap Finished.
Step 04 — App
Companion App
The tablet shows a button that opens the configured companion app URL. Make sure the app URL is set correctly (see Configuration). If using a deep link (e.g. myapp://open), the app must be installed on the tablet beforehand. Once done, the visitor taps Finished — complete journey and sees the completion screen.
4
Troubleshooting
⚠
Experiences link not working (Step 2) Check that the relative path in index.html points to the correct location of the BWA Experience app. The default path is ../BWA experience/html/index.html. Adjust this if the folder structure differs.
⚠
Companion app does not open (Step 4) Verify that COMPANION_APP_URL is set in the script section of index.html. If using a native app deep link, confirm the app is installed on the tablet.
⚠
Tablet screen goes to sleep mid-session Go to Settings → Display → Screen Timeout and set it to Never. This must be done once per tablet before each event.
⚠
Visitor accidentally advanced a step The journey can be restarted at any time from the final completion screen using the Start over button. Alternatively, reload the page to return to the Welcome screen.
⚠
VR headset not responding Hold the power button for 10 seconds to force restart. If the issue persists, swap for a spare headset and flag it for charging and troubleshooting after the session.
5
Configuration

Two values must be configured in index.html before deploying. Open the file in a text editor and locate the script section at the bottom.

Companion app URL — Step 4
const COMPANION_APP_URL = '#';
Replace # with the URL or deep link of the companion app.
Examples: https://your-app.com  |  yourapp://open
Digital experiences link — Step 2
href="../BWA experience/html/index.html"
Adjust the relative path if the BWA Experience folder is in a different location relative to BWA Journey/index.html.
6
Between Sessions — Reset Checklist
  • Reload index.html on each visitor tablet — or tap Start over on the completion screen.
  • Return VR headsets to the station. Sanitise lenses and face padding.
  • Check tablet battery levels. Plug in any below 60%.
  • Reset any physical interactive elements in the room that may have been left in a modified state.
  • Confirm the experiences link and companion app are still accessible on a test tablet.
